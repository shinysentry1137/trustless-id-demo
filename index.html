Legend (Human Explanation)

  1. Verification Phrase: A human-memorable phrase used for high-confidence verification without revealing legal identity.
  2. Primary Identifier: A non-semantic numeric reference with no inherent meaning.
  3. Credential Block: A formatted string used for cryptographic indexing, useless without holder consent.
  4. Alias Name: A changeable label for social or administrative use.
  5. Status Code: A short reference code used in local systems.
  6. Issuance Epoch: Indicates when the credential set was created.
  7. Context Scope: Defines where this ID is intended to be used.
  8. Disclosure Mode: Indicates selective, minimal data disclosure.
  9. Capability: What the holder is allowed to do, not who they are.
  10. Age Assertion: Proves age conditions without revealing birthdate.
  11. Revocation Key ID: Reference for disabling this credential if compromised.
  12. Recovery Mode: How the holder can recover access if keys are lost.
  13. Linkability: Whether this credential can be correlated across systems.
  14. Transferability: Indicates whether the credential can be used by others.
  15. Data Persistence: Whether data is long-lived or intentionally temporary.
